# Дашборд мониторинга приёмной кампании

React + Express приложение для показа статистики заявок на большом телевизоре.

## Что внутри

- `React + Vite` — frontend.
- `Express` — backend-прокси к 1С, чтобы браузер не ходил напрямую в 1С и не ловил CORS/Basic Auth ошибки.
- `Recharts` — графики и диаграммы.
- `react-datepicker` — календарь для выбора конкретного дня, недели, месяца или года.
- `lucide-react` — крупные иконки для ТВ-интерфейса.
- Демо-режим с mock-данными, чтобы интерфейс открывался даже без доступа к 1С.

## Выбор диапазона на дашборде

В верхнем блоке можно переключить отображение: всё, день, неделя, 2 недели, месяц, год.
Для режимов «День», «Неделя», «2 недели», «Месяц» и «Год» появляется календарь:

- день — фильтрация строго по выбранной дате;
- неделя — календарная неделя с понедельника по воскресенье;
- 2 недели — 14 дней, включая выбранную дату;
- месяц — выбранный календарный месяц;
- год — выбранный календарный год.

## Настройка подключения к 1С

1. Скопировать файл настроек:

```bash
cp .env.example .env
```

2. Заполнить `.env`:

```env
PORT=3001
ONE_C_URL=http://10.205.100.22/yniv/hs/exchange_sayt/get_applicants_statistics
ONE_C_USER=http_user_reserv
ONE_C_PASSWORD=ваш_пароль
ONE_C_METHOD=GET
ONE_C_TIMEOUT=30000
ONE_C_USE_MOCK=false
```

В присланном тестовом proxy использовался пользователь `http_user_reserv`. В документе с описанием сервиса указан `http_user_mlgu`. Нужно оставить того пользователя, который реально проходит авторизацию на стороне 1С.

Если `.env` не создан или пароль не заполнен, backend автоматически отдаст demo/mock-данные.

## Запуск разработки

```bash

```

После запуска:

- frontend: `http://localhost:5173`
- backend: `http://localhost:3001`
- health-check: `http://localhost:3001/api/health`

## Production-запуск

```bash
npm install
npm run build
npm start
```

В production Express отдаёт собранный frontend из папки `dist` и обслуживает `/api/*`.

## Как работает proxy

Frontend делает запрос только на свой backend:

```js
POST /api/applicants-statistics
{
  "period": "2025-01"
}
```

Backend делает запрос в 1С с Basic Auth и передаёт в 1С тело:

```json
{
  "period": "2025-01"
}
```

Метод по умолчанию — `GET`, потому что так было реализовано в присланном рабочем proxy-примере. При необходимости можно поменять на `POST` через переменную `ONE_C_METHOD=POST`.

## Подготовка к показу на телевизоре

Откройте frontend в браузере на ТВ/мини-ПК и включите полноэкранный режим:

- Windows/Chrome/Yandex Browser: `F11`;
- масштаб браузера: 100–125%;
- лучшее разрешение: Full HD или 4K.

Дашборд автоматически обновляет данные каждые 30 минут. Также есть ручная кнопка «Обновить».
